Tween - gsap.to().from().fromTo()

A Tween is what does all the animation work - think of it like a high-performance property setter. You feed in targets (the objects you want to animate), a duration, and any properties you want to animate and when its playhead moves to a new position, it figures out what the property values should be at that point applies them accordingly.

Image Tag with simple to() Tween

        
let tweenBball1 = gsap.to(".basketballImg1",
  {
    duration: 3,
    x: document.querySelector('.svg-container').offsetWidth,
    backgroundColor: "coral",
    borderRadius: "20%",
    border: "3px solid black",
    ease: "power3.out",
    paused: true,                              // if true, the animation will pause itself immediately upon creation
    clearProps: "x,backgroundColor,border"     // clear from the elementâ€™s style property when the tween completes
  }
);

SVG Element using yoyo property with rotation

  • ease: "back.inOut(1)" is used to give the animation a slight rocking motion
  • repeat: -1 in combination with yoyo: true creates an infinite forward-backward animation
        
gsap.set(".seesaw", { rotation: -30, transformOrigin: "50% 50%" });
gsap.to(".seesaw", {
    duration: 4,
    rotation: 30,
    ease: "back.inOut(1)",
    yoyo: true,
    repeat: -1
});

Timeline - gsap.timeline(..)

A powerful sequencing tool that acts as a container for tweens and other timelines, making it simple to control them as a whole and precisely manage their timing. Without Timelines, building complex sequences would be far more cumbersome because you'd need to use a delay for every animation

Sequencing Patterns

  • Relative Sequence: "+=1" or "-=1", meaning start relative to the previous tween ending

    Note that if any durations on tweens are updated, this pattern would not account for it and also have to be updated. For example, say a square and circle tween both have a duration of 1 and both start 1 second after a star tween because the square tween has a relative sequence of "+=1". If the duration on the square gets updated to 3 seconds, then the circle tweens relative sequence value of "-=1" would have to get updated to "-=3" in order for it to start at the same time as the square tween. This is due to the fact that the last tween's placement (the circle tween) is relative to the previous tween.

  • Start or End of most recent: "<[N]" for start or ">[N]" for end, meaning start at the start or end of the most recently-added animation, where N is an optional number that pushes the animation out.

    If you want 1-N tweens to start or end at the same exact time based on a previous animation, this is the pattern to use. This pattern does reduce the need for sequence Labels. 

                  
// insert 1 second after the START of the most recently added animation
tl.to(".class", {x: 100}, "<1");

// insert 1 second after the END of the most recently added animation
tl.to(".class", {x: 100, duration: 1}, ">1");

  • Absolute Sequence: "1" or "[N]", meaning start directly at the N second mark

Simple timeline sequencing using tweens and a timeline label (position parameter)

        
var tl1 = gsap.timeline( { paused: true });
tl1.from(".basketballImg3", { duration: 1.5, opacity: 0, scale: 0.3, ease: "back" });
tl1.to(".basketballImg3", { duration: 1, rotation: 360 }, "<.5");
tl1.from("#circles2 .circle", { duration: 1, opacity: 0, y: 150, stagger: 0.25 });
tl1.addLabel("circlesOutro", "+=1");
tl1.to("#circles2 .circle", { duration: 0.5, opacity: 0, x: 300, ease: "power3.out" }, "circlesOutro");
tl1.to(".basketballImg3", { duration: 0.5, opacity: 0, x: -300, ease: "power3.out" }, "circlesOutro");

Easing through colors

        
let tl2 = gsap.timeline( { paused: true, repeat: -1 });
let colors = [ "#FF7F00", "#FFFF00", "#00FF00", "#0000FF", "#2E2B5F", "#8B00FF" ];
for (let i = 0; i < colors.length; i++) {
    let color = colors[i];

    tl2.to("html", { "--rainbowColor": color, duration: 2 });
}
tl2.play();
